QubitPage® Quantum OS

The world's first web-based quantum operating system, creating a seamless bridge between quantum hardware, artificial intelligence, and human operators.

What is QubitPage OS?

QubitPage OS is a complete quantum computing platform that runs in your browser. It provides:

  • Quantum Circuit Lab — Write, compile, and simulate quantum circuits using QPlang
  • Quantum Oracle Game — The world's first quantum prediction game
  • Quantum Crypto Tools — Encryption, QRNG, BB84 key distribution
  • ARIA AI Assistant — AI-powered quantum computing guide (Groq + Gemini)
  • Full Desktop Environment — Windows, taskbar, apps, themes

Quick Start

1. Open Circuit Lab

Double-click the Circuit Lab icon on the desktop, or use the Start Menu.

2. Write Your First Circuit

# Bell State — quantum entanglement
qreg q[2]
creg c[2]

H q[0]
CNOT q[0], q[1]

measure q -> c

3. Run the Simulation

Click ▶ Run to simulate on the Stim engine. You'll see results like:

|00⟩  ████████████ 50.1%
|11⟩  ████████████ 49.9%

4. Try the Terminal

Open the Terminal and try these commands:

simulate bell          # Run a Bell state
simulate ghz           # Run a GHZ state
simulate grover        # Run Grover's search
backends               # List IBM Quantum backends
qrng 256               # Generate quantum random numbers
hash Hello World       # Post-quantum hash
aria What is a qubit?  # Ask the AI

5. Play the Quantum Game

Open Quantum Oracle and test your quantum intuition by predicting measurement outcomes!

System Architecture

Overview

QubitPage OS consists of layered modules, each serving a specific role in the quantum computing stack.

Kernel (quantum_kernel.py)

The kernel wraps QPlang, Stim, and IBM Quantum into a unified API. It handles:

  • QPlang compilation (source → QASM + Stim)
  • Stim circuit simulation (Bell, GHZ, Grover, custom)
  • IBM Quantum hardware execution
  • Quantum game round generation

AI Agent (ai_agent.py)

ARIA uses a priority-chain of AI providers:

  1. Groq SDK — LLaMA 3.3-70B (fastest)
  2. Gemini SDK — Gemini 2.0 Flash (fallback)
  3. Groq REST — Direct HTTP API (last resort)

QPlang Compiler

The QPlang pipeline:

Source (.qpgl) → Lexer → Parser → AST → Type Checker → Compiler → QASM + Stim

QBP Transpiler

The QBP pipeline:

Source (.qbp) → QBP Lexer → QBP Parser → Document Tree → HTML Transpiler → HTML/CSS/JS

Quantum Error Correction

Steane Code [[7,1,3]]

The core error correction unit. Encodes 1 logical qubit into 7 physical qubits, correcting any single-qubit error.

Parity check matrix H:
  X-stabilizers: S1 = IIIXXXX, S2 = IXXIIXX, S3 = XIXIXIX
  Z-stabilizers: S4 = IIIZZZZ, S5 = IZZIIZZ, S6 = ZIZIZIZ

Syndrome → Error mapping (O(1) lookup):
  000 → No error
  001 → Error on qubit 0
  010 → Error on qubit 1
  011 → Error on qubit 3  ...etc

Auto-Escalation

QPlang monitors error rates and automatically escalates protection:

LevelCodePhysical QubitsThreshold
0[[4,2,2]]4< 0.1% error
1[[7,1,3]] Steane7Default
2Surface d=317> 1% error
3Surface d=549> 2% error
4Surface d=797> 5% error

Decoders

  • Hamming Lookup — O(1) table for Steane code (7 syndromes)
  • MWPM — Minimum Weight Perfect Matching via PyMatching
  • Collision Clustering — Union-Find, FPGA-ready
  • Neural Decoder — CNN, ONNX exportable for edge deployment

Decoder Reference

Hamming Lookup (decoder_steane.py)

O(1) decoder for the Steane code. Maps 3-bit syndromes to correction operators via a pre-computed table.

SYNDROME_TABLE = {
    (0,0,0): None,          # No error
    (0,0,1): PauliOp("X", 0),  # q0
    (0,1,0): PauliOp("X", 1),  # q1
    (0,1,1): PauliOp("X", 3),  # q3
    ...
}

MWPM (decoder_mwpm.py)

Uses PyMatching with Stim's detector error model for optimal surface code decoding.

Collision Clustering (decoder_cc.py)

Software implementation of Union-Find for FPGA deployment. Processes syndrome clusters in near-linear time.

Neural Decoder (decoder_neural.py)

CNN-based decoder trained on Stim-generated syndrome data. Exportable to ONNX for hardware acceleration.

Quantum Crypto Tools

Quantum Encryption (OTP)

Uses CSPRNG for one-time pad key generation, providing information-theoretic security.

BB84 Protocol

Simulates the BB84 quantum key distribution protocol:

  1. Alice generates random bits and random bases (Z or X)
  2. Alice prepares qubits and sends them to Bob
  3. Bob measures with random bases
  4. Alice and Bob compare bases publicly
  5. Matching bases → shared secret key

QRNG

Quantum Random Number Generator using Stim simulation. Generates truly random bits from quantum superposition via Hadamard gates.

Post-Quantum Hash

SHA3-256 (Keccak) — resistant to Grover's algorithm with 128-bit post-quantum security.
